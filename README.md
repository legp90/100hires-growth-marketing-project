## 🔍 Troubleshooting (Problems Found & Solutions)

During the environment setup, the following critical technical challenges were identified and resolved:

* **Problem 1: Confusion Between the Web Dashboard and the Desktop App**
  * *Description:* Initially, I attempted to search for and install the extensions within the web browser interface (`cursor.com/dashboard`), which restricted access to the global Marketplace.
  * *Solution:* I identified that extension management must be performed natively within the local Windows application, which I accessed using the `Ctrl + Shift + X` shortcut.

* **Problem 2: Subscription Restriction in Claude Code**
  * *Description:* When attempting to log into Claude Code as requested, the Anthropic web interface indicated a technical restriction: a Pro, Max, or active API credits account is required.
  * *Solution:* In order to maintain project momentum and respect the strict 48-hour deadline, I chose to leave the extension installed, document the audit trail, and proceed with the rest of the portfolio setup.

* **Problem 3: Missing 'Git: Clone' Command in Cursor**
  * *Description:* When trying to link the repository, Cursor did not recognize the cloning command and the corresponding button was unresponsive.
  * *Solution:* I diagnosed that the laptop lacked the core Git software. I downloaded the official 64-bit installer for Windows, completed the setup, and restarted Cursor so the environment could recognize version control commands perfectly.
